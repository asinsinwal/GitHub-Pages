Q1. Explain some additional concerns related to using REST apis.
Solution:
a) REST apis work on HTTP protocol. Because of request/response protocol, it cannot support services initiated by server.
-> "Some of the  problems of REST arenâ€™t inherent problems of the architectural style but rather drawbacks of the REST over HTTP implementation."

b) According to some papers, PUT and DELETE aren't yet supported by all web browsers, thus it is considered okay to "overload POST" by adding a query string argument like method=PUT or method=DELETE on the URI being POSTed.

c) REST api calls are difficult to test or debug, because the developer / engineer has to check the URL, method used (like GET / POST) and status codes (if error response occurred) and thus it becomes a tedious job to handle.
